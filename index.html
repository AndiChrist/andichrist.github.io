Create default Java application

August 19, 2017 by Andi

If you want to create a very basic Java application from scratch with Gradle, you can write

gradle init --type java-application

in your shell.

You'll receive the following directory structure:

            .
            ├── build.gradle
            ├── gradle
            │   └── wrapper
            │       ├── gradle-wrapper.jar
            │       └── gradle-wrapper.properties
            ├── gradlew
            ├── gradlew.bat
            ├── settings.gradle
            └── src
                ├── main
                │   └── java
                │       └── App.java
                └── test
                    └── java
                        └── AppTest.java

To build and run simply run the application with the created Gradle wrapper

./gradlew run

or with "plain" Gradle

gradle run

As an successful outcome you should get the following output:

> Task :run
Hello world.


BUILD SUCCESSFUL in 4s
2 actionable tasks: 2 executed

Use different Gradle version

August 19, 2017 by Andi

You can use a different version of Gradle if you want, or if there are some requirements. Simply create the Gradle wrapper with the following command:

gradle wrapper --gradle-version 2.0

You can do this in combination with the init command:

gradle wrapper --gradle-version 2.0 init --type java-application

You can check the current Gradle wrapper's version with

./gradlew -version
